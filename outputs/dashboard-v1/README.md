# Приемная кампания 360: dashboard v1

Первая версия сделана как самодостаточный web-прототип утвержденного дашборда. Сейчас данные демонстрационные и лежат в `src/data.js`; интерфейс уже разделен так, чтобы позже заменить этот файл на внутренний API университета.

## Как открыть

Откройте `index.html` в браузере или запустите папку на любом локальном web-сервере.

## Как развивать до онлайн-версии

1. Внутренний сервер университета принимает данные из приемной комиссии, CRM Bitrix, мероприятий и рекламных кабинетов.
2. Данные нормализуются по уникальному коду абитуриента и хранятся внутри периметра университета.
3. Локальная нейросеть получает только внутренние данные, формирует сегменты, риск недобора, следующее лучшее действие и персональные тексты.
4. Дашборд читает агрегаты и списки задач через закрытый API.
5. Bitrix получает готовые персональные сообщения и задачи для менеджеров без передачи чувствительных данных наружу.

## Минимальная архитектура

- Frontend: статический web-интерфейс, который можно развернуть на внутреннем сервере.
- Backend API: FastAPI или Node.js внутри инфраструктуры университета.
- Storage: PostgreSQL для абитуриентов, касаний, заявлений, согласий и коммуникаций.
- AI layer: локальная LLM + правила скоринга, без облачных отправок персональных данных.
- Audit: журнал действий, версий рекомендаций и отправленных сообщений.
